# SpringBoot 微服务概述

- 为什么使用微服务？

  使用微服务对复杂的业务系统进行拆分，按业务模块功能进行拆分，形成一个个单独的微服务进行管理，便于持续交付与集成，便于系统维护，更好的容错与系统故障隔离。

- 微服务有哪些不好？

  1、分布式系统当出现问题时，定位故障点非常困难，须通过链路调用才能分析到，问题具体出现在那个节点上。
  
  2、服务之间的通信会出现高延迟与数据一致的问题。
  
  3、服务数量比较多，部署与管理工作量较大
  
  4、系统稳定性下降
